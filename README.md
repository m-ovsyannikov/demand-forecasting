# Прогнозирование спроса на товары

Учебный ML-проект: прогноз ежедневного спроса по историческим данным о продажах.
Задача регрессии на временных рядах с группировкой по товарам.

Выполнено в рамках учебной практики (1 курс, «Прикладная информатика», профиль
«Прикладной искусственный интеллект», НГУ). Роль — ML-инженер: подготовка данных,
обучение моделей, оценка качества.

## Подход

- **Признаки:** лаги (`lag_1`, `lag_7`), календарная сезонность (день недели, месяц),
  one-hot кодирование товаров.
- **Разбиение train/test — по времени** (всё до 2024-10-01 в train), чтобы модель
  не «подсматривала» в будущее. Для временных рядов случайный сплит некорректен.
- **Модели:** от наивных бейзлайнов к нейросети.
- **Метрики:** MAE и RMSE на едином тестовом срезе.

## Результаты

| Модель                  | MAE    | RMSE   |
|-------------------------|--------|--------|
| Наивный прогноз (lag_1) | 13.27  | 17.78  |
| Сезонный прогноз (lag_7)| 11.92  | 16.59  |
| Линейная регрессия      | 8.99   | 11.37  |
| **MLPRegressor**        | **7.63** | **9.73** |

Лучший результат — у MLP (2 скрытых слоя 64/32, ReLU, Adam, early stopping,
StandardScaler в Pipeline). Но главный вывод: простые бейзлайны оказались сильными
конкурентами, и сложную модель нельзя считать хорошей, пока она не обошла их на
корректной валидации.
